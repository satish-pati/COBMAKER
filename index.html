COBMAKER – Interactive Block-Based Programming Environment for Legacy Language Learning and Real-Time Code Generation


Tool Description

COBVIS is an innovative, browser-based learning environment designed to teach COBOL—a foundational legacy programming language—using modern visual programming principles. Targeting both novice programmers and experienced developers transitioning to legacy system maintenance, COBVIS leverages intuitive drag-and-drop interfaces, hierarchical section visualizations, and real-time code generation to bridge the gap between modern learning styles and the structured syntax of COBOL. COBVIS transforms traditionally tedious COBOL education into an engaging, interactive experience. Inspired by the success of visual programming platforms like Scratch and Blockly, COBVIS is the first comprehensive visual learning platform built specifically for COBOL, empowering the next generation of developers to sustain and innovate on critical legacy infrastructure.

  • Windows 8/10/11, macOS 12+, or Linux (Ubuntu 20.04+)
  • Processor: Intel Pentuim, Core i3/i5/i7/i9 or Ultra, AMD Ryzen 5 or better
  • RAM: Minimum of 4GB

The software was tested on a Windows 11 system with 16 GB RAM.
Software:
  • Node.js version 16 or higher
  • React (via react-scripts 5.0.1)
  • Python version 3.10+
  • Flask (Python backend framework)
  • GnuCOBOL (for COBOL code compilation)
  • Blockly (Visual block-based programming library)
  • react-split-pane (for UI layout)
  • Axios (for frontend-backend communication)

API Tokens and Accesses
  • GitHub Finegrained PAT
  • Vercel Account for accessing backend API endpoint. Be sure to mail your signed-up email to cs23m104@iittp.ac.in with the purpose of using the tool for gaining access. This is for security reasons.

Usage

The tool consists of a frontend built using React and a backend powered by Flask. Follow the steps below to run COBMAKER locally on your machine:

  1. Clone the GitHub repository by running the following command:
    git clone https://github.com/satish-pati/COBMAKER.git
    cd COBMAKER
  2. Install Frontend Dependencies:
    cd client
    npm install
  3. Install Backend Dependencies:
    • Navigate to the server directory and create a virtual environment:
    • Note for Windows users: The backend is supported only in Linux environments. You must use WSL (Windows Subsystem for Linux) with Ubuntu or any other supported distribution.
    • Launch your WSL terminal (e.g., Ubuntu) and navigate to the backend directory:
      • cd ../server
      python3 -m venv venv
    • Activate the virtual environment:
      • source venv/bin/activate
    • Install the required Python packages:
      • pip install -r requirements.txt
  4. Start the Flask server (backend):
    python3 app.py
  5. Start the React frontend:
    cd ../client
    npm start
  6. Access the Tool:

    Open a Chromium-based browser and go to the following address:

    http://localhost:3000

Once both servers are running, you can start using COBMAKER to visualize and assess COBOL code interactions.

Approach diagram

The age metrics are directly retrieved by the extension JavaScript, popup.js, from the GitHub API, as this is a lightweight task. URL validation and extraction of the owner and repository names are also handled by 'popup.js'. For the complexity metrics, a back-end service written in Python and deployed on Vercel clones the repository into a local directory. Python functions are used to calculate the LOC and CCN scores. The lizard library is used to compute CCN. The results are then returned to the extension as a JSON object, where the final calculations are performed, and the scores are displayed to the user.

Goal

Helps learners and organizations rapidly onboard new COBOL developers by providing an interactive, block-based visual environment that generates real-time COBOL code. Manually teaching COBOL through text-based tutorials is slow and error-prone, often deterring modern learners. With COBVIS, both novice and experienced programmers can drag and drop intuitive blocks to explore COBOL syntax, receive immediate compilation feedback, and build working programs—significantly reducing the time and effort required to understand and maintain critical legacy systems.
