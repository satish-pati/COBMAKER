LegacyLens

LegacyLens: Legacy Scoring for GitHub Repositories


Tool Description

LegacyLens was designed as a light-weight chrome extension to assess GitHub repositories in real time to evaluate the legacy state of the repository, with the aim of helping the developers make a quick informed decision about the future of the repository. The assessment is done through analyzing two key categories of metrics - 1.) age of the repository and 2.) code complexity metrics. These metrics provide information about the repository which is not explicitly mentioned in the existing documentation.

Usage

The tool is designed as a Chrome extension. The extension can be installed by following the below steps:

  1. Clone the GitHub repository by running the following command:
    git clone https://github.com/Chiranjeevi2001/legacylens.git
  2. Provide the GitHub Personal Token for rate limit extension
    • Create your GitHub PAT (Leave permissions as default)
    • Copy the token and paste it in the RHS of line number 4 in `popup.js` file:
      const token = 'PASTE YOUR TOKEN HERE'
  3. Open Chrome Extensions manager
    • Open Google Chrome or any Chormium-based browser
    • Navigate to the extensions page by entering chrome://extensions in the address bar
  4. Enable Developer Mode: In the top right corner, toggle the Developer Mode switch to enable it
  5. Load the extension
    • Click the Load Unpacked button
    • In the file browser, select the folder where the repository was cloned (it should contain `manifest.json` file)
    • Click Open
    • Click on the pin icon to pin the extension to your browser's toolbar
  6. Enjoy the extension!: Run the extension on the main page of any public GitHub repository to see the assessment results

Approach diagram

The age metrics are directly retrieved by the extension JavaScript, popup.js, from the GitHub API, as this is a lightweight task. URL validation and extraction of the owner and repository names are also handled by 'popup.js'. For the complexity metrics, a back-end service written in Python and deployed on Vercel clones the repository into a local directory. Python functions are used to calculate the LOC and CCN scores. The lizard library is used to compute CCN. The results are then returned to the extension as a JSON object, where the final calculations are performed, and the scores are displayed to the user.

Goal

Helps developers and open-source contributors in assessing the legacy state of large repositories. This task, if done manually, is time-consuming, which will slow down the contribution rate. However, if there was a quicker and automated way to know the state of the repository, it would allow the developers to make quick decisions instead of spending their time manually inspecting the repository.
